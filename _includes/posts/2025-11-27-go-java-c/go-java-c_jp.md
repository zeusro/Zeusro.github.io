

2013年、私はすでにブログ园（cnblogs）に登録しており、最初に使用した職業プログラミング言語は C# でした。

当時はまだ多少のアクセスがありました。[张善友](https://www.cnblogs.com/shanyou) 氏は Microsoft C# の中国エバンジェリストとして知られており、彼がまだ Tencent 内で苦戦していた頃、私は少しだけ信じかけたこともありました。しかし、最終的には `Go` 言語へと乗り換えました。

2015年の頃、私はインターネット上で java boy（~~JB~~）たちとよく口論していました。今思えば少し滑稽でもあります。

2020年、科兴科学園のエレベーターで偶然彼を見かけました。私は彼に向かって「今の C# ってもうダメなんじゃないの？」と直接言いました。彼は非常に気まずそうな顔をしていましたが、同じ下りエレベーターに乗っていたため、礼儀として軽く返事してくれました。

## 形式論理と定義

**皮パン原理（Leather Pants Principle）**：  
他人の質問に答えるとき、必ずしも真実だけを語る必要はない。多くの場合、相手が思い込みたい答えを与えたほうが効果的なことがある。なぜなら、多くの人は自分が理解できないものを受け入れる勇気がなく、未知を理解しようともしないからである。人は大抵、既に持っている概念で世界を解釈しようとする。

**AOT Dynamic Library**（Ahead-of-Time Dynamic Library）とは、.NET の AOT コンパイルモードによって生成されるネイティブ動的リンクライブラリ（.dll / .so / .dylibなど）の一種であり、C/C++製ライブラリと同じように他の言語・プラットフォームから呼び出せ、.NET ランタイムを必要としない。

## 背景

　　.NET（以下 N）の心の声：「今どき企業システムで .NET を使うところは減っている。でも少なくとも文法の美しさと IDE の強さは宇宙最強でしょ？」

　　JAVA（以下 J）の心の声：「我らが JAVA は天下無双、Android、Kotlin、Scala、とにかく ―― 最強。」

## 炎上の始まり

![image](/img/in-post/go-java-c/go-java-c.png)

N が Android を学び始めたことが発端でした。少し触ってみたものの Android Studio の不具合に文句を言い始めました。あの IDE に問題が多いのは周知の事実です。そして N は Google にも文句を言いはじめ、最終的には Java をも巻き添えにしてしまいました。公平を期すために N の不満を挙げます。

1. プロジェクトが無理由でフリーズ
1. エミュレーターの縦横回転バグ
1. Dalvik VM が低性能で笑いもの
1. Java に暗黙的型推論がない

J: 「.NETって何に使うの？大規模クラスタは無理だし… 開発者も少ない… しかもライセンス料？」

N: 「MySQL も使えるし、Java が流行ったのは統合ソリューションが早かっただけ」

J: 「当時 .NET はオープンソースじゃなかったからね。それがオープンソースの力。最後は .NET も開かざるを得なかった」

N（心）：「それ関係ないだろ…」  
口では：「stackoverflow.com は25台のサーバーで動いてる。クラスタ弱いって偏見だろ。技術力の問題だ。」

J: 「本当に強いなら JD（京东）は Java に移行してないよ」

N: 「それは言語設計の問題で性能とは無関係！」

J: 「それも含めて言語設計。嫌なら使うな。Java8 には syntax sugar があるし、Scala にもある」

N: 「Java8 に暗黙的型推論あるの？」

J: 「syntax sugar の意味わかってる？」

N: 「`var` も syntax sugar だろ」

J: 「そういう書き方が好きなら Kotlin 使えば？Bytecode は Java と同じ。オープンソースの利点。たとえ .NET が真似しても Java の優位は揺るがない」

N（心）：「お前1人で勝てないから Kotlin 呼ぶなよ… .NET だって MSIL あるぞ… MSBuild だってクロスプラットフォームだぞ…」  
でもプライドのために言ったのは：「まるで Java がラムダを先に発明したかのように言うなよ」

J: 「まるで Java の VM が .NET と違うとでも？」

通りすがり: 「.NET が好きなら Android なんて学ぶなよ」

N は危機を悟り撤退：  
「良い悪いは比較の中にしかない。どちらも長所短所がある。批判のない社会に進歩はない。政府が気に入らないなら、川に飛び込むのか？生きるしかないだろ。以上、解散」

J、止まらず追撃：  
「Java コミュニティで Java をけなすとか礼儀ゼロ。しかも Java の理解浅いのに文句？Android VM も知らないくせに？」

**はい。ボコられていたのは私です。  
最後のセリフに答えると：Dalvik VM が微妙なのはニュース見れば分かる。だから Google は ART を作った。欠点に触れられたくないのは心が狭い。もちろん .NET にだって欠点はたくさんある。でも指摘されたら認める。それだけ。  
この言葉はここにだけ書く。なぜなら皮パン原理を思い出したから。**

彼らは極端に自信過剰だった。

## 2025-11-27 からの考察

以上は `2015-04-27 23:48` にブログ园へ投稿した記事をまとめたものです。  
今見ると、博客园はほぼ崩壊状態、張善友氏は深圳友浩达を創業し、今も Microsoft MVP。  
TIOBE の指標によれば、C# は Java のシェアを徐々に奪いつつある。  
今や .NET Core は AOT Dynamic Library として完全なネイティブコードへコンパイルでき、以下の言語から利用可能：

- C/C++
- Go
- Rust
- Python（FFI）
- Unity Native Plugin
- iOS / Android Native

C++ 視点で言えば、.NET Core で C++ コードを書けるようなもの。これは完全に「裏取り」です。  
Microsoft はプログラミング界で最も邪悪な組織 No.1 に相応しい。

そして最近、新しい namespace も追加されている。素晴らしい：

- `IAsyncEnumerable` ― 非同期データストリーム
- `System.Threading.Channels` ― Go の channel を「参考に」
- `EFCore.BulkExtensions` ― ORM バルク書き込み
