## Microsoft любит принимать изменения

Возможно, Microsoft просто любит принимать изменения. Каждые несколько лет они любят выпускать целый новый набор технологических фреймворков.

1. .NET framework
1. .NET core
1. Visual Basic
1. F#

- web

- desktop

- Mobile

С точки зрения синтаксиса, C#—самый элегантный язык, который я когда-либо видел.
[LINQ намного лучше stream API Java 8](http://www.zeusro.com/2018/03/08/linq-vs-stream/).

Но Microsoft с самого начала пошла по неправильному пути, выбрав тесную привязку `C#` к платформе `Windows`. План их CEO был таков: укрепить нашу позицию через тесную привязку и установить доминирование.

Люди голосуют ногами. Будь то `Windows` или `Windows Server`, лицензирование—значительная стоимость. Microsoft в то время была слишком сосредоточена на краткосрочной выгоде, что привело к крупному поражению C#.

Поэтому `Xamarin` пошел по другому пути, предложив кроссплатформенную реализацию с .NET.

Позже Microsoft глубоко задумалась и решила принять open source. Но даже после приобретения `Xamarin` путь интеграции был полон трудностей.

Когда Microsoft объявила .NET 5 на конференции Build 2019, они четко заявили: "В будущем будет только один .NET, и вы сможете использовать его для таргетинга Windows, Linux, macOS, iOS, Android, tvOS, watchOS и WebAssembly, и многое другое." Microsoft объявила в апреле, что когда был выпущен preview 2, он уже обрабатывал 50% трафика на их сайте .NET.

Но с 2002 по 2019 год эти 17 лет были в основном потрачены впустую. Каждая итерация технологий Microsoft была в основном несовместима с предыдущими версиями. Для обычных разработчиков это было настоящей пыткой.

Так что в основном технологический стек Microsoft остался только с несколькими направлениями:

1. Разработка игр Unity 3D
1. Работа в Microsoft
1. Быстрая разработка низкозатратных приложений без обслуживания
